# Security Policy

## Avvertenza generale

I file Microsoft Office possono contenere contenuti malevoli, macro, collegamenti esterni, oggetti incorporati o strutture anomale. L'analisi di file sconosciuti deve avvenire con prudenza.

## Raccomandazioni operative

- Analizzare file non fidati in ambiente isolato o dedicato.
- Usare copie controllate invece degli originali quando possibile.
- Limitare la propagazione dei campioni sospetti.
- Evitare di aprire i file con suite Office produttive se non necessario.

## Segnalazione vulnerabilita

Se ritieni di aver individuato una vulnerabilita nel progetto:

- non pubblicare exploit dettagliati direttamente nelle issue;
- invia una descrizione tecnica minima, l'impatto osservato e i passaggi riproducibili in modo prudente;
- attendi coordinamento prima della divulgazione pubblica.

In assenza di un canale dedicato, apri una issue senza allegare campioni sensibili e senza includere dati personali o informazioni riservate.

## Divieto di allegare contenuti sensibili

Nelle issue pubbliche non devono essere allegati:

- file Office reali;
- evidenze digitali;
- documenti contenenti dati personali;
- campioni riservati o coperti da segreto, confidenzialita o restrizioni di diffusione.
